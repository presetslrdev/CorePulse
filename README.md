# CPU Monitor Notifier

Трей-приложение для Windows 11, которое следит за нагрузкой **отдельных ядер** процессора.

## Идея

Стандартные мониторы показывают общую загрузку CPU, но не замечают ситуацию, когда одно ядро
надолго «залипло» под 100% (зависший однопоточный процесс, майнер, разогнавшийся фоновый сервис) —
общая загрузка при этом может выглядеть безобидно (на 16 ядрах это всего ~6%).

CPU Monitor Notifier:

- 📊 показывает **информативную иконку в трее** — мини-диаграмма нагрузки всех ядер с цветовой
  кодировкой (зелёный / жёлтый / красный);
- 🔔 выдаёт **toast-уведомление**, если ядро (или несколько) находится под нагрузкой выше порога
  дольше заданного времени (настраивается, можно отключить);
- 🕵️ определяет и показывает **процесс-виновник**, грузящий ядро;
- ⚙️ настройки: порог %, длительность, cooldown уведомлений, автозапуск.

```powershell
dotnet build
dotnet run --project src/CpuMonitorNotifier
```

Публикация одним exe:

```powershell
dotnet publish src/CpuMonitorNotifier -c Release -r win-x64 --self-contained false
```

## Использование

- Иконка в трее — «живая», обновляется ~8 раз в секунду и во главу угла ставит нагрузку
  **самого горячего ядра** (крупным числом и цветом: зелёный < 60%, жёлтый 60–90%, красный > 90%).
- Стиль иконки выбирается в настройках:
  - **Кольцо + %** — кольцевой индикатор + число (по умолчанию, самый читаемый на малом размере);
  - **Сегментное кольцо** — сегмент на каждое ядро, самое горячее подсвечено, его % в центре;
  - **Спидометр** — шкала-тахометр на 270°;
  - **Жидкость + %** — контейнер наливается до уровня нагрузки с анимированной волной;
  - **Сетка кругов** — обзор всех ядер, круг «наливается» по нагрузке.
- Tooltip: самое нагруженное ядро, общая загрузка и самый прожорливый процесс.
- Двойной клик по иконке или пункт меню «Настройки…» — стиль иконки, порог, длительность,
  cooldown, вкл/выкл уведомлений, интервал опроса, автозапуск.
- Пункт меню «Проверить уведомление» — показать тост по текущему горячему ядру
  (удобно проверить, что уведомления не блокируются режимом «Фокусировка внимания»).
- Уведомление содержит номера ядер, длительность нагрузки и top-3 процессов-кандидатов;
  кнопка «Диспетчер задач» открывает taskmgr.

## Как определяется виновник

Windows не предоставляет статистику «процесс × ядро» без ETW (нужны права администратора),
поэтому используется эвристика: раз в секунду снимаются дельты CPU-времени всех процессов,
и при алерте показываются процессы, чьё потребление сопоставимо с числом нагруженных ядер.
Для типового сценария — зависший однопоточный процесс держит одно ядро на 100% — эвристика
практически точна. Подробнее: [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md).

## Диагностика

Если приложение сообщает об ошибке счётчиков производительности, пересоберите их кэш:
`lodctr /R` в консоли администратора.
